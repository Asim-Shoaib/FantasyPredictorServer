## Quick Start

> **Prerequisite:** Python 3.11+, Node 18+, and the [data files](#required-data-files) in place.

```bash
# 1. Install Python dependencies (from the src/ directory)
pip install -r requirements.txt

# 2. Install frontend dependencies
cd frontend && npm install && cd ..

# 3. Start both servers
#    Terminal A вҖ” Flask API on http://localhost:5000
python run.py

#    Terminal B вҖ” Vite dev server on http://localhost:5173
cd frontend && npm run dev
```

Open **http://localhost:5173** вҖ” the frontend proxies all `/api/*` calls to the Flask backend automatically.

> **One-command option:** `python run.py --with-frontend` starts both servers in a single terminal.

## What is DraftGenius?

DraftGenius is a full-stack web application that generates optimised PSL fantasy cricket teams for a selected match-up. Pick two franchises, and the system produces three distinct XI recommendations вҖ” **Safe**, **Explosive**, and **Balanced** вҖ” in seconds, complete with live Genetic Algorithm progress streamed to the browser.

Three AI/ML techniques work together to score every player:

- **Hidden Markov Model (HMM) form prediction** вҖ” classifies each player's current form as *cold*, *average*, or *hot* and outputs transition probabilities for their next appearance.
- **Elo rating** вҖ” tracks each player's relative strength chronologically across eight T20 leagues (PSL, IPL, BBL, BPL, CPL, SA20, ILT20, LPL) and converts it into a per-match score multiplier clamped to [0.75, 1.40].
- **Genetic Algorithm optimiser** вҖ” searches the legal team space (budget, role counts, franchise limits) and returns the best XI for each of three risk strategies.

The React frontend streams live GA progress via Server-Sent Events so you can watch fitness converge generation by generation.

## Installation

### 1. Clone the repository

```bash
git clone <repo-url>
cd <repo>/src
```

### 2. Install Python dependencies

```bash
pip install -r requirements.txt
```

### 3. Install Node dependencies

```bash
cd frontend && npm install && cd ..
```

### 4. Prepare data files

The server needs at minimum:

- `data/all_leagues_player_match_elo.csv` вҖ” master ELO dataset (~35 MB, generated by the pipeline)
- `models/hmm_form_models.joblib` вҖ” pre-trained general HMM artifact
- `data/psl_2026_roster_overrides.json` вҖ” PSL player roster

See [Data Pipeline](#data-pipeline) below to build the ELO CSV from raw Cricsheet match JSONs.

## Running in Development

### Option A вҖ” two terminals (recommended)

```bash
# Terminal 1: Flask backend (port 5000)
python run.py

# Terminal 2: Vite dev server (port 5173, proxies /api вҶ’ port 5000)
cd frontend && npm run dev
```

Open `http://localhost:5173` in your browser.

### Option B вҖ” single terminal

```bash
python run.py --with-frontend
```

### run.py flags

| Flag | Default | Description |
|---|---|---|
| `--port PORT` | `5000` | Flask listen port |
| `--with-frontend` | off | Also launch `npm run dev` in `frontend/` |
| `--update-data` | off | Download latest Cricsheet data and rebuild ELO CSV before starting |
| `--retrain-models` | off | Delete cached per-player HMM models so they re-fit on first predict |

## API Overview

The backend exposes a REST + SSE API under `/api/`. A complete reference вҖ” including all request/response shapes, examples, and error codes вҖ” is in [`backend/BACKEND.md`](backend/BACKEND.md).

| Method | Path | Description |
|---|---|---|
| `GET` | `/api/health` | Server liveness check |
| `GET` | `/api/match/teams` | List PSL franchises |
| `GET` | `/api/players` | Player profiles (optionally filtered by two teams) |
| `GET` | `/api/players/<id>/history` | Last 20 match records for a player |
| `GET` | `/api/constraints` | Current GA constraint settings |
| `POST` | `/api/constraints` | Save constraint overrides |
| `POST` | `/api/generate-teams` | Synchronous GA вҖ” returns all three teams |
| `POST` | `/api/generate-teams-start` | Start async GA, returns `run_id` |
| `GET` | `/api/stream/<run_id>` | SSE stream of GA progress + final result |
| `GET` | `/api/evolution/<run_id>` | Per-generation evolution snapshots |
| `POST` | `/api/roster/bench/<player_id>` | Bench a player |
| `POST` | `/api/roster/reinstate/<player_id>` | Reinstate a player |
| `GET` | `/api/roster/status` | All bench overrides |
| `POST` | `/api/admin/fetch-photos` | Fetch ESPN headshots for all roster players |
| `GET` | `/api/admin/photo-status` | Photo coverage counts |

## Data Pipeline

The ELO CSV is built offline from Cricsheet-style match JSON archives. You only need to re-run it when you want to ingest new match data.

### Quick rebuild (no download)

```bash
python -m backend.cli
```

### Download + rebuild

```bash
python -m backend.cli --update-data
```

Or via the launcher:

```bash
python run.py --update-data
```

### Pipeline stages

1. **Download** (`CricsheetUpdater`) вҖ” fetches ZIP archives from Cricsheet (ETag-aware; skips unchanged files), unpacks new match JSONs into `data/<league>_male_json/`.
2. **Parse** (`MatchParser` + `SmartMatchCache`) вҖ” converts each Cricsheet match JSON to per-player fantasy-point records. Already-parsed matches are read from `.cache/final_results_cache.parquet` and never re-parsed, making incremental runs fast.
3. **Score** (`Pipeline.run()`) вҖ” applies batting / bowling / fielding fantasy scoring rules.
4. **Form features** (`FormFeatureBuilder`) вҖ” adds rolling average and streak columns.
5. **Elo** (`EloEngine`) вҖ” computes chronological per-player Elo ratings across all leagues and writes the final `data/all_leagues_player_match_elo.csv`.
6. **Photos** (`ProfileFetcher`, optional with `--fetch-photos`) вҖ” fetches ESPN CDN headshot URLs and writes `data/output/player_profiles.json`.

### Supported leagues

`psl`, `ipl`, `bbl`, `bpl`, `cpl`, `sa20`, `ilt20`, `lpl`

### Useful flags

```bash
# PSL only, last 3 years
python -m backend.cli --leagues psl --cutoff-years 3

# Force full re-run even if no new files
python -m backend.cli --force-pipeline

# Download + rebuild + fetch photos
python -m backend.cli --update-data --fetch-photos
```

Add `data/people.csv` (downloaded from [cricsheet.org/downloads](https://cricsheet.org/downloads/)) for player photo lookups вҖ” it maps Cricsheet identifier to ESPN cricinfo ID.
